More appreciation for a for-loop
program transformationrecursionimperative programmingfunctional programmingfor-loopsmutual recursionoptimizationalgorithmsj.n. oliveiracontinuation-passing styledefunctionalizationcalculational designcomputer science
  • Core Idea: Presents a method by J.N. Oliveira to mechanically transform systems of non-tail, primitive recursive equations into efficient, imperative for-loops.
  • Key Advantages:
    • The transformation is direct and calculational.
    • It avoids complex intermediate steps like Continuation-Passing Style (CPS) and defunctionalization.
    • It requires no specific properties (e.g., associativity) of the operations involved.
  • Core Technique (Mutual Recursion Law):
    • A system of equations like: 
      f(n+1) = s(f(n), g(n))
g(n+1) = t(f(n), g(n))
    • Can be converted into a for-loop where the loop's state is the tuple (f(n), g(n)), updated at each step by the functions s and t.
  • Primary Examples:
    • Subtractorial: A function like factorial but using non-associative subtraction (n - subt(n-1)), demonstrating the method's generality where traditional accumulator transformations fail.
    • Cosine Approximation: A more complex derivation for the Taylor series expansion of cos(x), requiring four mutually recursive equations that are transformed into a single for-loop with four state variables.
