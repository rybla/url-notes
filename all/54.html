http://www.e-pig.org/downloads/notanum.pdf
  • Problem: Standard representations for syntax with binders are problematic.
    • Using only names requires complex and error-prone α-conversion to prevent variable capture.
    • Using only de Bruijn indices makes code unreadable and unreliable due to complex index-shifting arithmetic.
  • Solution: A hybrid representation is proposed.
    • Free variables (those bound in the current context) are represented by names, allowing for simple, arithmetic-free manipulation.
    • Bound variables (those bound within a sub-expression) are represented by de Bruijn indices, providing a canonical form that avoids α-conversion.
  • Core Technique: The system uses a "perspective shift" when traversing into binders, inspired by Huet's zippers.
    • When a program navigates under a binder (e.g., into a function body), the bound variable is locally converted from an index to a named, free variable for manipulation within that new scope.
    • This makes the implementation of syntactic constructions (like generating induction principles) significantly simpler and safer.
  • Data Structure Example (Haskell): 
    data Expr = F Name       -- Free variable
          | B Int        -- Bound variable (de Bruijn index)
          | Expr :$ Expr   -- Application
          | Expr :→ Scope  -- Universal quantification
