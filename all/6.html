googleapis/js-genai
  • SDK: @google/genai for TypeScript/JavaScript.
  • Target APIs: Gemini Developer API & Vertex AI.
  • Prerequisites: Node.js v20+. Vertex AI requires a configured Google Cloud project with the Vertex AI API enabled and gcloud authentication.
  • Installation: npm install @google/genai
  • Initialization:
    • Instantiate GoogleGenAI.
    • Use apiKey for Gemini Developer API.
    • Use vertexai: true, project, and location for Vertex AI.
    • Supports environment variables for configuration (GOOGLE_API_KEY or GOOGLE_GENAI_USE_VERTEXAI).
  • Core API Modules:
    • ai.models: Core generation methods (generateContent, generateContentStream).
    • ai.chats: Stateful multi-turn conversation management.
    • ai.caches: Caching for large prompt prefixes to reduce cost.
    • ai.files: File uploading and referencing for large, reusable assets.
    • ai.live: Real-time, multi-modal sessions (text, audio, video).
  • Features:
    • Streaming: generateContentStream for chunked responses.
    • Function Calling: Define functions via FunctionDeclaration (name, description, JSON schema) and pass them as tools to the model.
    • Model Context Protocol (MCP): Experimental support for integrating MCP servers as tools.

Is Chain-of-Thought Reasoning of LLMs a Mirage? A Data Distribution Lens
published: 2025-08-02

Chain-of-Thought (CoT) prompting has been shown to improve Large Language Model (LLM) performance on various tasks. With this approach, LLMs appear to produce human-like reasoning steps before providing answers (a.k.a., CoT reasoning), which often leads to the perception that they engage in deliberate inferential processes. However, some initial findings suggest that CoT reasoning may be more superficial than it appears, motivating us to explore further. In this paper, we study CoT reasoning via a data distribution lens and investigate if CoT reasoning reflects a structured inductive bias learned from in-distribution data, allowing the model to conditionally generate reasoning paths that approximate those seen during training. Thus, its effectiveness is fundamentally bounded by the degree of distribution discrepancy between the training data and the test queries. With this lens, we dissect CoT reasoning via three dimensions: task, length, and format. To investigate each dimension, we design DataAlchemy, an isolated and controlled environment to train LLMs from scratch and systematically probe them under various distribution conditions. Our results reveal that CoT reasoning is a brittle mirage that vanishes when it is pushed beyond training distributions. This work offers a deeper understanding of why and when CoT reasoning fails, emphasizing the ongoing challenge of achieving genuine and generalizable reasoning.
