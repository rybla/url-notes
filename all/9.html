  • Two Notions of Diversity: The article connects two independently developed mathematical concepts of "diversity":

    • Leinster-Cobbold (LC) Diversity: A family of measures D_q^Z(p) quantifying the "effective number of elements" in a set X with a similarity function Z, given a probability distribution p. The maximum diversity D(X) is the maximum of this value over all p.
    • Bryant-Tupper (BT) Diversity: A function δ on the finite subsets of a set X that generalizes the notion of a metric, satisfying δ(A) = 0 for singletons and δ(A∪B) ≤ δ(A∪C) + δ(C∪B).

  • Main Result: Maximum LC diversity gives rise to a BT diversity.

    • Theorem 1: For a metric space X, the function δ(A) = D(A) - 1 defines a BT diversity on its finite subsets. log D(A) also defines a BT diversity.

  • Key Properties: The BT diversity derived from maximum LC diversity exhibits unique properties distinct from previously studied examples like the diameter.

    • Subadditivity (Thm 2): D(⋃ A_i) ≤ ∑ D(A_i).
    • Fractional Subadditivity (Thm 3): The complexity per element D(X)/n is bounded by the average complexity per element of its n-1 sized subsets.
    • Superlinearity (Thm 4): In ℝ, log D is superlinear (δ(A+B) ≥ δ(A)+δ(B)), contrasting with the sublinearity of diameter. This property generalizes the Cauchy-Davenport and Brunn-Minkowski inequalities in one dimension.

Levels of Autonomy for AI Agents
artificial intelligenceai agentsai autonomylevels of autonomyhuman-ai interactionai governanceai safetyai regulationai evaluationautonomous systems
  • Core Thesis: Argues that an AI agent's level of autonomy should be a deliberate design decision, distinct from its intrinsic capabilities or operational environment.
  • Autonomy Framework: Proposes a five-level framework for agent autonomy, defined by the user's role:
    • L1 (Operator): User has full control and planning responsibility; the agent assists on demand.
    • L2 (Collaborator): User and agent share planning and execution, delegating tasks to each other.
    • L3 (Consultant): Agent leads planning and execution, consulting the user for high-level guidance and feedback.
    • L4 (Approver): Agent operates mostly independently, only pausing for user approval on predefined consequential actions or when blocked.
    • L5 (Observer): Agent is fully autonomous; the user can only monitor and use an emergency stop.
  • Governance Proposal: Introduces "autonomy certificates," a proposed governance mechanism where a third-party body certifies an agent's maximum operational autonomy level based on an evidence-based "autonomy case" provided by the developer.
  • Evaluation Methodology: Suggests "assisted evaluations" to measure autonomy. This method determines the minimum level of user involvement required for an agent to successfully complete a task, thereby classifying its operational autonomy level.
